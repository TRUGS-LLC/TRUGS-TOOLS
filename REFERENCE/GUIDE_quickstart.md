# TRUGS Tools — Quickstart

⏱️ **Time: ~15 minutes** | Python 3.11+ | Single runtime dependency (`trugs-store`)

---

## What is TRUGS?

TRUGS (Tree-Relational Unified Graph Structures) is a protocol for representing structured data as typed graphs with nodes, edges, and dimensions. TRUGS Tools is a Python toolchain for creating, validating, rendering, and managing TRUG files. It supports 10 branches (Python, Rust, LLVM, Web, Writer, Semantic, Orchestration, Living, Knowledge, Executable) and ships with CLI commands and a Python API.

## Installation

```bash
cd TRUGS_TOOLS
pip install -e .
```

For development (includes test dependencies):

```bash
pip install -e ".[test]"
```

Verify the install:

```bash
tg validate --help
```

## Your First TRUG

Generate a minimal Python TRUG using the CLI:

```bash
tg generate --branch python --template minimal --output my_first.json
```

This creates a valid TRUG with 3 nodes (MODULE → FUNCTION → STATEMENT). Inspect it:

```bash
cat my_first.json
```

You should see JSON with `nodes`, `edges`, `dimensions`, and `capabilities` fields.

Save a complete example with more nodes and extensions:

```bash
tg generate --branch python --template complete --output my_complete.json
```

## Validate a TRUG

### CLI

```bash
tg validate my_first.json
```

Expected output:

```
✓ my_first.json: Valid TRUG (3 nodes, 0 edges)
```

Validate multiple files at once:

```bash
tg validate my_first.json my_complete.json
```

Get JSON output for scripting:

```bash
tg validate --format json my_first.json
```

### Python API

```python
from trugs_tools import validate_trug

result = validate_trug("my_first.json")

if result.valid:
    print("Valid!")
else:
    for error in result.errors:
        print(error)
```

You can also validate a dict directly:

```python
import json

with open("my_first.json") as f:
    trug = json.load(f)

result = validate_trug(trug)
print(result.valid)  # True
```

## Explore TRUG Information

Use `tg info` to inspect a TRUG's structure:

```bash
tg info my_first.json
```

Output includes metadata, statistics, hierarchy depth, and node type distributions.

Show only metadata:

```bash
tg info --section metadata my_first.json
```

Get a one-line summary:

```bash
tg info --format compact my_first.json
```

Get machine-readable output:

```bash
tg info --format json my_first.json
```

## Render Documentation

The renderer compiles a `folder.trug.json` into three deterministic markdown files:

| Output | Audience |
|--------|----------|
| **AAA.md** | Developers + LLMs — 8-phase development tracker |
| **README.md** | Humans — orientation and contents |
| **ARCHITECTURE.md** | LLMs + Architects — full node/edge details |

### Render all three

```bash
tg render folder.trug.json
```

### Preview without writing files

```bash
tg render folder.trug.json --dry-run
```

### Render a single file type

```bash
tg render folder.trug.json --file-type architecture
```

### Output to a different directory

```bash
tg render folder.trug.json -o /tmp/output
```

Same input always produces byte-identical output — no LLM, no inference, pure template engine. All generated files are marked with `<!-- GENERATED BY TRUGS — DO NOT EDIT -->`.

## Filesystem Commands

TRUGS filesystem commands manage your project graph alongside the actual filesystem. All commands work with `folder.trug.json` as the single source of truth.

### Initialize a project

```bash
trugs tinit . -n "My Project" --scan
```

Creates `folder.trug.json` with a FOLDER root node. `--scan` discovers existing files and adds them automatically.

### Add files to the graph

```bash
trugs tadd main.py utils.py
trugs tadd data.csv -t CONFIGURATION
```

File types are inferred from extensions (`.py` → SOURCE, `.md` → DOCUMENT, `.json` → CONFIGURATION).

### List files with graph metadata

```bash
trugs tls .
trugs tls . --edges
```

### Create typed edges between nodes

```bash
trugs tlink main_py utils_py -r DEPENDS_ON
trugs tlink test_py main_py -r TESTS
```

### Find nodes by type or pattern

```bash
trugs tfind -t SOURCE
trugs tfind -n "\.py$"
```

### Sync graph with filesystem

```bash
trugs tsync .              # Full sync — discover new files, detect removed ones
trugs tsync . --dry-run    # Preview changes
```

### Other commands

| Command | Purpose |
|---------|---------|
| `trugs tcd <node>` | Navigate graph by node ID |
| `trugs tmove <node> --name new.py` | Atomic rename — file + graph updated together |
| `trugs tdim list` | List dimensions |
| `trugs twatch .` | Watch for changes, auto-regenerate docs |

See [SPEC_filesystem.md](SPEC_filesystem.md) for the full reference.

## Working with TRUGs Programmatically (trugs-store)

For programmatic graph access, use the `trugs-store` library instead of raw `json.load`/`json.dump`:

```bash
pip install -e TRUGS_STORE/trugs-store
```

```python
from trugs_store import JsonFilePersistence, InMemoryGraphStore

# Load a TRUG
persistence = JsonFilePersistence()
store = persistence.load("folder.trug.json")

# Query
print(store.node_count(), store.edge_count())
tasks = store.find_nodes(type="TASK", status="OPEN")
children = store.get_children("root_node_id")

# Traverse
for node, edge, depth in store.traverse("start_id", max_depth=3):
    print(f"{'  ' * depth}{node['id']} --[{edge['relation']}]-->")

# Mutate (bidirectional invariant enforced automatically)
store.add_node(new_node, parent_id="parent_id")
store.mark_stale("old_node_id", "file deleted")
store.add_edge({"from_id": "a", "to_id": "b", "relation": "DEPENDS_ON"})

# Validate against TRUGS CORE
violations = store.validate_graph()

# Save
persistence.save(store, "folder.trug.json")
```

All folder-* CLI tools use trugs-store internally. See `TRUGS_STORE/SPEC_844_graphstore_protocol.py` for the full 22-method Protocol.

## Python API

### Validate

```python
from trugs_tools import validate_trug

result = validate_trug("my_trug.json")
print(result.valid)    # True/False
print(result.errors)   # List of error messages
```

### Generate

```python
from trugs_tools import generate_trug

# Minimal TRUG
trug = generate_trug("python", template="minimal")

# With extensions
trug = generate_trug("python", extensions=["typed", "scoped"])

# Save to file
from trugs_tools.generator import generate_to_file
generate_to_file("output.json", branch="python", template="complete")
```

### Render

```python
from trugs_tools.renderer import render_aaa, render_readme, render_architecture, render_all

# Render individual files (returns string)
aaa_content = render_aaa("folder.trug.json")
readme_content = render_readme("folder.trug.json")
arch_content = render_architecture("folder.trug.json")

# Render all three (returns dict)
results = render_all("folder.trug.json")
# {"AAA.md": "...", "README.md": "...", "ARCHITECTURE.md": "..."}

# Render and write to disk
render_all("folder.trug.json", output_dir=".")
```

### End-to-end example

```python
import json
from trugs_tools import generate_trug, validate_trug

# Generate
trug = generate_trug("python", template="complete", extensions=["typed"])

# Validate
result = validate_trug(trug)
assert result.valid

# Save
with open("example.json", "w") as f:
    json.dump(trug, f, indent=2)

print(f"Created TRUG with {len(trug['nodes'])} nodes, {len(trug['edges'])} edges")
```
